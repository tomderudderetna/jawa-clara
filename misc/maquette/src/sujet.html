Aide : Rendu avec SVN

Les rendus doivent être effectués via Subversion.

Pour commencer à développer il faut récupérer une copie du répertoire de rendu en local sur votre machine.

Cela se fait à l'aide de la commande:

						$> svn checkout --username votre_login "$$RENDU$$"

Vous devez utiliser votre login et votre mot de passe d'intra pour vous connecter au SVN.


Pour ajouter un fichier au Subversion vous devez utiliser la commande svn add.

Exemple:

						$> svn add test.c

Pour envoyer vos fichiers sur le serveur vous devez utiliser la commande svn commit.

						$> svn commit -m "message_de_commit"

Une fois votre rendu prêt, vous pouvez demander la validation via l'interface web.

Jour01

C'est une moulinette qui ramassera et qui évaluera votre rendu. Pensez donc à vérifier que vous avez respecté *scrupuleusement* le sujet.

Seule cette page servira de référence, donc pas de "Mais tel asset a dit que..."

La moulinette arrête de donner des points au premier exercice faux ;)

Les exercices devront être rendus dans des dossiers et fichiers distincts dont les noms sont donnés dans les énoncés. Exemple pour l'exercice 1 : $$RENDU$$/closed/closed.sh

Vous ne devez pas rendre votre fichier passwd, il sera fourni par la moulinette lors de la correction

Pour les scripts il faut appeler le fichier passwd comme s'il était dans le répertoire courant (exemple : "cat passwd")
